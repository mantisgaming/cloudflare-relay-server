Server notes

When a client joins, the relay first sends the server a `con` packet with a numeric `pid`. This demo can automatically accept that peer by sending `con` back to the relay, or you can do it manually from the peer list.

Use the reconnect button after closing the server socket to verify that the relay issues a replacement reconnect key and replays currently connected peers.
